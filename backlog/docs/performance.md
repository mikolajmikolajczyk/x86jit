## Tiering — wall-clock by mode

`eager` compiles every block on first execution; `tier` interprets a block until it is hot (50 runs) then compiles it inline; `bg` compiles hot blocks on a worker thread (`x86jit-run` ships `tier`); `region-bg` (BGT-6, opt-in) tiers a hot loop up to a background-compiled superblock region — a win only on long multi-block warm loops (`hotloop`), a loss on one-shot workloads. `best↓` is the fastest mode's speedup over `eager`.
